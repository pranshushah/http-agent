- By default **http-pro** throws an error if [response.status](https://developer.mozilla.org/en-US/docs/Web/API/Response/status) is not between `200` and `299`.

<DefaultError />
<br />
<br />

:::note
**_http-pro_** throws a custom Error called `HttpProError` when `statucCode` is not valid. you can check if given error is `HttpProError` or not by using `isHttpProError` function.
:::

<br />

## Custom-error handling

- If you want to customize the default behaviour of error logic, you can use [validateStatus](../API/HttpOptions#validatestatus) property of `httpOptions`. it will get `statusCode` as an arguement and it needs to return `boolean`, if function returns `true` it will not throw an Error otherwise it will.
